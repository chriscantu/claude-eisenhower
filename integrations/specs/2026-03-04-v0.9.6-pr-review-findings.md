## 🔴 Critical Issues — Must Fix Before Merge (5)

### C1 · `hooks/hooks.json` — No "today" anchor for date comparisons

**Agents**: Code Reviewer (unique find)

The SessionStart hook instructs the LLM to compare `Scheduled:` and `Check-by:` dates
against "today" but never defines what today is. `currentDate` is injected via CLAUDE.md
system context, but a SessionStart hook may fire before the main conversation context is
loaded. The LLM may use its training cutoff date or hallucinate one — causing overdue items
to be missed or false positives every session.

**Fix**: Embed the date reference explicitly in the prompt:
```
Use today's date as provided in your session context (currentDate field).
Only count tasks whose date is strictly on or before today's date (YYYY-MM-DD comparison).
```

---

### C2 · `hooks/hooks.json` — Missing field handling produces silent miscounts

**Agents**: Code Reviewer + Silent Failure Hunter (2-way confirmation)

The hook says "Scan Active tasks for any with a `Scheduled:` date" but most Active tasks
have no `Scheduled:` field (it's only written by `/schedule`). No instruction for absent
fields → LLM may treat missing = overdue, flooding the summary with false positives.
Same issue for `Check-by:` on Delegated tasks. Malformed dates (e.g. "TBD", "next week")
also have no handling.

**Fix**: Add explicit skip rules to the hook prompt:
```
- Skip Active tasks with no Scheduled: field (they have not been scheduled yet).
- Skip Delegated tasks with no Check-by: field.
- If a date value is not a valid YYYY-MM-DD, skip that task and append:
  "[N] task(s) have unparseable date values."
- If the Active or Delegated section is absent or empty, skip that clause entirely.
```

### C3 · `hooks/hooks.json` — Pluralization inconsistency corrupts LLM output

**Agents**: Comment Analyzer

The template says `"Overdue: [N] active task(s)"` (with literal `(s)`) but the example
says `"Overdue: 1 active task"` (no `(s)`). LLMs weight examples over templates — for
N > 1 the LLM has no example and may produce `"2 active task(s)"` literally.

**Fix**: Provide both singular and plural examples:
```
Example (single): 'Task Board: 2 Inbox, 3 Active, 1 Delegated, 8 Done.
  Overdue: 1 active task. Check-in due: 1 delegation.'
Example (multiple): 'Task Board: 2 Inbox, 3 Active, 2 Delegated, 8 Done.
  Overdue: 2 active tasks. Check-in due: 2 delegations.'
```

### C4 · `skills/skill-enhancer/SKILL.md` — Self-enhancement HALT is not terminal

**Agents**: Silent Failure Hunter + Final Code Reviewer (2-way confirmation)

The self-enhancement guard (M9 fix) is a dangling bullet placed OUTSIDE the decision
table, after a closing code fence. An LLM reading linearly may display the HALT message
and fall through to `**Exit**: Environment confirmed. Proceed to Phase 1.`

There is no explicit "stop here" directive — the HALT is ambiguous about whether it
terminates the phase or is just a conditional warning.

**Fix**: Move the guard inside the numbered decision table as an explicit row with a
terminal directive:
```markdown
   - Target artifact is `skills/skill-enhancer/SKILL.md` → **HALT**.
     Say: "Self-enhancement is not supported — the skill cannot improve its own
     operating rules mid-session. Use the skill-creator skill to modify skill-enhancer."
     **Stop here. Do not reach the Exit line. Do not proceed to Phase 1.**

Any HALT above is terminal. Do not read the Exit line below.
```

### C5 · `commands/setup.md` — `expandedPath` is an undefined pseudovariable

**Agents**: Comment Analyzer (unique find)

The AppleScript snippet uses `quoted form of expandedPath` but `expandedPath` is a
prose concept from the preceding step — it is never defined as an AppleScript variable.
On paths containing `~` this will produce broken osascript at runtime.

**Fix**: Make the substitution explicit:
```applescript
set thePath to do shell script "echo " & quoted form of userProvidedPath
do shell script "test -d " & quoted form of thePath & "/scripts && echo exists || echo missing"
```
Or add a note: *"Substitute `expandedPath` with the literal expanded path (with `~`
resolved to the home directory) before calling osascript."*

## 🟡 Important Issues — Should Fix (5)

### I1 · `commands/setup.md` — Unbounded retry loop, no shell failure path

**Agents**: Silent Failure Hunter

The path validation loop has no retry cap and no escape hatch. A user who provides a
wrong path repeatedly, or runs setup in a sandboxed environment where `do shell script`
fails, has no way out except closing the session.

**Fix**:
- Add retry cap: "Allow up to 3 attempts."
- Add escape: "After 3 failures, tell the user they can type 'skip' to configure
  `plugin_root` manually in `integrations/config/task-output-config.md`."
- Add shell failure path: "If the shell command returns an error (not 'exists' or
  'missing'), say: 'Could not verify the path due to a system error. Enter the path
  manually and I'll trust your input, or type skip.'"

### I4 · All command files — Memory fallback: no directory creation, no user notification

**Agents**: Silent Failure Hunter + Comment Analyzer (confirmed)

All 6 fallback blocks say "create the file if it doesn't exist" but not "create the
`memory/` directory." If the directory is absent the write fails silently. Additionally,
the user is never told the fallback was used — CONNECTORS.md claims "A warning note will
appear at each call site" but no such note exists in the actual blocks.

**Fix** at each of the 6 call sites (schedule.md ×2, execute.md ×2, delegate.md ×1,
SKILL.md ×1):
```
If the productivity:memory-management skill is not available:
1. Notify the user: "Note: memory-management skill not found. Logging locally to
   memory/stakeholders-log.md."
2. Ensure the memory/ directory exists before writing (create if absent).
3. If the write fails: "Could not record this follow-up ([reason]). Track it manually."
```

**Bonus fix in execute.md**: The fallback in the "offer to log" block (Step 3.7) fires
regardless of whether the user accepted the offer. Scope it: "If the user accepted the
offer AND the skill is not available, log locally."
